# Нефункциональные требования: аналог Trello

## Из чего состоит приложение
- Backend (серверная часть) работает как REST API на Spring Boot: Spring Web, Spring Security, Spring Data JPA
- Frontend (то, что видит пользователь в браузере) сделан на Vue 3 и обращается к backend только через REST API в формате JSON
- Backend строится по слоям: Controller принимает запросы, Service содержит бизнес-логику, Repository работает с базой данных
- База данных PostgreSQL. Изменения структуры базы хранятся в виде миграций Liquibase, а не правятся руками

## Безопасность
- Вход в систему через токен (JWT). Сервер не хранит сессии, токен присылается в каждом запросе
- Пароли хранятся не в открытом виде, а в виде хэша (BCrypt)
- Открыть доску и её содержимое может только владелец и приглашённые участники, проверка прав идёт на сервере
- Все данные, которые приходят от пользователя, например из форм, проверяются на сервере перед сохранением
- Настроен CORS, чтобы frontend с другого адреса мог обращаться к backend

## Скорость работы
- Обычные операции (создать, посмотреть, изменить карточку) отвечают быстрее 300 мс на локальном окружении
- Списки (доски, карточки, комментарии) при большом количестве данных подгружаются частями (пагинация), а не всё сразу

## Надёжность
- У всех ошибок API единый формат ответа: код ошибки, текст сообщения, время
- Сервер отвечает правильными HTTP-кодами: 200 успех, 400 ошибка в запросе, 401 не авторизован, 404 не найдено и так далее
- Перемещение карточки или колонки это единая операция. Если что-то пошло не так, данные не остаются наполовину изменёнными

## Тестирование и поддержка кода
- Сервисный слой (бизнес-логика) покрыт unit-тестами (JUnit 5 + Mockito)
- Основные REST-эндпоинты проверены интеграционными тестами
- API описан документацией через Swagger, чтобы можно было посмотреть список эндпоинтов в браузере

## Развёртывание (как приложение запускается)
- Backend упаковывается и запускается через Docker
- Настройки для разных окружений (разработка, прод) хранятся отдельно в `application.yml`
- Опционально: через docker-compose можно одной командой поднять backend, базу данных и frontend вместе

## Удобство использования
- В интерфейсе можно перетаскивать карточки и колонки мышкой
- Интерфейс адаптивный и одинаково удобен как на компьютере, так и на телефоне
- Приложение корректно работает в последних версиях Chrome, Firefox, Edge
