# Пояснения к диаграммам

## Файлы в этой папке
- `диаграмма-сценариев-использования.drawio`: общая use case диаграмма, актёры и все сценарии системы
- `диаграмма-деятельности.drawio`: блок-схемы для трёх сценариев с ветвлениями "Создать доску", "Пригласить участника", "Создать карточку" (по вкладке на каждый)

## Use case диаграмма

### Актёр
Человечек слева. Обозначает роль, которая взаимодействует с системой: Гость, Пользователь, Владелец доски.

### Сценарий использования
Овал. Одно конкретное действие, которое актёр выполняет в системе, например "Создать доску".

### Простая линия (без стрелки)
Association, связь актёра со сценарием: этот актёр может выполнить этот сценарий.

### Стрелка с пустым треугольником (генерализация)
Означает "является частным случаем". У нас: Владелец доски является частным случаем Пользователя, то есть умеет всё то же самое, что и обычный пользователь, плюс что-то ещё (приглашать участников, удалять доску).

### Пунктирная стрелка «включает» (include)
Сценарий A **всегда** обязательно использует сценарий B как свою часть, без исключений. Например, "Создать доску" всегда сначала проверяет права доступа, это не опция, а обязательный шаг. "Проверить права доступа" вынесен в отдельный пузырь, чтобы не повторять этот шаг в описании каждого сценария, который его использует.

Аналогия: как приватный метод `checkAccess()` в коде, который вызывают несколько публичных методов, не потому что это опция, а потому что без него нельзя.

### Пунктирная стрелка «дополняет» (extend)
Сценарий A **может, но не обязан** дополнить сценарий B при определённых условиях. Базовый сценарий B полностью работает и без A. Например, "Добавить метку" дополняет "Редактировать карточку": можно отредактировать карточку и вообще не трогать метки.

Правило, чтобы не путать include и extend: если убрать связанный пузырь, а базовый сценарий сломается, это include. Если базовый сценарий продолжит работать как ни в чём не бывало, это extend.

### Цвета сценариев
- Голубой: доступно гостю, до входа в систему
- Зелёный: доступно любому участнику доски
- Оранжевый: доступно только владельцу доски
- Серый: внутренняя проверка, не отдельное действие пользователя (переиспользуемый через include шаг)

## Диаграмма деятельности

### Чёрный кружок
Начало сценария.

### Кружок с кольцом (двойная окружность)
Конец сценария (одно из возможных завершений: успешное или через ошибку).

### Скруглённый прямоугольник
Действие (шаг сценария).

### Прямоугольник с двойной рамкой по бокам (shape=process)
Вызов другого сценария, аналог include с use case диаграммы. Например "Проверить права доступа" внутри "Пригласить участника".

### Ромб
Точка ветвления. Из ромба выходит несколько стрелок с подписями ("да" / "нет"), каждая ведёт к своему продолжению сценария в зависимости от условия.
