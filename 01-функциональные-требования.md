# Функциональные требования: аналог Trello

## Действующие лица (кто пользуется системой)
- **Гость.** Человек, который ещё не вошёл в систему. Ему доступны только регистрация и вход.
- **Пользователь.** Человек, который вошёл в систему. Может создавать свои доски и работать в досках, куда его пригласили.
- **Владелец доски.** Пользователь, который создал конкретную доску. У него больше прав: может удалить доску, приглашать и удалять участников.

## Требования

### 1. Вход и профиль
- Пользователь может зарегистрироваться по email и паролю
- Пользователь может войти в систему по логину и паролю
- Пользователь может выйти из системы
- Пользователь может посмотреть и изменить своё имя в профиле

### 2. Доски
- Пользователь может создать доску (название, описание)
- Пользователь может просмотреть список своих досок (где он владелец или участник)
- Пользователь может открыть доску и увидеть все колонки и карточки
- Владелец может редактировать название и описание доски
- Владелец может удалить доску
- Владелец может пригласить участника на доску по email
- Владелец может удалить участника с доски
- Участник может просмотреть список участников доски
- Владелец может назначить участнику роль: только просмотр или полное редактирование

### 3. Колонки
- Участник может создать колонку на доске
- Участник может переименовать колонку
- Участник может менять порядок колонок, перетаскивая их мышкой
- Участник может архивировать колонку вместо удаления, карточки внутри архивируются вместе с ней

### 4. Карточки
- Участник может создать карточку в колонке (указать название)
- Участник может открыть карточку и увидеть все подробности
- Участник может изменить название и описание карточки
- Участник может перетащить карточку в другую колонку или на другое место мышкой
- Участник может архивировать карточку вместо удаления
- Участник может назначить одного или нескольких исполнителей из участников доски
- Участник может указать срок выполнения задачи
- Участник может добавить или убрать метку у карточки
- Участник может оставлять комментарии к карточке
- Участник может добавить в карточку чек-лист с подзадачами и отмечать подзадачи выполненными
- Участник может прикрепить файл к карточке

### 5. Метки
- Участник может создать метку на доске (название + цвет)
- Участник может удалить метку с доски

### 6. Поиск и фильтрация
- Участник может искать карточки по названию в рамках доски
- Участник может фильтровать карточки по меткам, по исполнителю или по обоим сразу

### 7. Уведомления и история
- Пользователь получает уведомление, если его назначили исполнителем карточки
- Участник может открыть журнал изменений карточки: кто, когда и что поменял
